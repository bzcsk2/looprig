# 0618 完成项

## fix(core): scope supervisor loop tools by workflow phase

### 问题
`supervisor_analyse` 阶段 Supervisor 拿到 `read_file`/`grep`，导致自我探索循环，永远不执行 `transition("worker_do")`，Worker 不被调动。

### 修改文件

| 文件 | 变更 |
|------|------|
| `packages/core/src/resolve-effective-tools.ts` | 新增 `WorkflowPhase` 类型导入；用 `SUPERVISOR_LOOP_ANALYSE_TOOLS`/`CHECK`/`INTERVENE`/`DEFAULT` 四组 phase 集合替换单 `SUPERVISOR_LOOP_TOOLS`；新增 `supervisorLoopToolsForPhase()` 辅助函数；`resolveEffectiveTools()` 改用 phase-scoped 过滤；`ResolveEffectiveToolsOpts` 新增 `workflowPhase` 字段 |
| `packages/core/src/engine.ts` | `submit()` 签名新增 `workflowPhase?: WorkflowPhase`；用 `buildSupervisorLoopModePrompt()` 生成 phase 特化的 system prompt（analyse/check/intervene 各有独立指引）；新增 `phaseMaxTurns` 限制（analyse 2 turn, intervene 1 turn）；透传 `workflowPhase` 到 `resolveEffectiveTools()` |
| `packages/core/src/dual-agent-runtime/runtime.ts` | `submit()` 签名新增 `workflowPhase?: WorkflowPhase`；透传给 `engine.submit()` |
| `packages/core/src/workflow-coordinator/coordinator.ts` | `runSupervisorAnalyse()`/`runWorkerDo()`/`runWorkerReport()`/`runSupervisorCheck()`/`runSupervisorIntervene()` 各自传入当前 phase 给 `submit()` |

### Phase 工具集

| Phase | 工具 |
|-------|------|
| `supervisor_analyse` | `get_goal`, `list_dir` |
| `supervisor_check` | `get_goal`, `list_dir`, `read_file`, `grep` |
| `supervisor_intervene` | `get_goal` |
| 默认（无 phase） | `get_goal` |

- `bun run typecheck` — 通过
- `bun test packages/core` — 1206 全通过，0 fail

## fix(tui): bound transcript history and render window for long sessions

### 问题
TUI 在长时间会话中 transcript/timeline 无界增长，UI 每次变更全量投影/全量渲染，导致内存占用上升、GC 压力增大、Ink/React reconcile 成本上升，表现为 TUI 越跑越卡。

### 修改文件

| 文件 | 变更 |
|------|------|
| `packages/tui/src/store/transcript-store.ts` | 新增 `TranscriptStoreStats`、`TranscriptTrimOptions`、`getStats()`；round-aware `trimToLimitInternal()`（按 `roundId` 分组，组内任一 streaming/running 则整组跳过）；`canTrimEntry()`/`canTrimGroup()`/`getTrimGroupId()` 辅助方法；`bumpAfterMutation()`（trim + bump 合并）；所有结构性方法换用 `bumpAfterMutation()`；流式方法（`appendPartDelta`/`setTextPart`/`finalizePart`）保持纯 `bump` 不触发 trim |
| `packages/tui/src/store/transcript-reader.ts` | 新增 `TranscriptReaderStats` 和 `getStats()` |
| `packages/tui/src/store/bridge-runtime.ts` | 新增 `getStats()` 返回 `warningsLength`/`messageQueueLength` |
| `packages/tui/src/store/index.ts` | 导出新类型 |
| `packages/tui/src/bridge.tsx` | 新增 `MAX_WARNINGS=100`、`MAX_MESSAGE_QUEUE=50`、`appendBoundedQueue()` 辅助函数；所有 `warnings` append 加 `.slice(-MAX_WARNINGS)`；所有 `messageQueue` append 换 `appendBoundedQueue()`（满时丢最旧） |
| `packages/tui/src/DeepiMessages.tsx` | 提取 `getVisibleTimeline()` 纯函数；`DEFAULT_RENDER_WINDOW=300` + `useMemo` 窗口化渲染 + 隐藏历史提示 |

### 新增/修改测试

| 文件 | 测试内容 |
|------|----------|
| `packages/tui/__tests__/transcript-store.test.ts` | `getStats` 正确性验证 |
| `packages/tui/__tests__/bridge-runtime.test.ts` | `getStats` 返回队列长度 |
| `packages/tui/__tests__/deepi-messages.test.ts` | `getVisibleTimeline` 纯函数 5 场景（限内不变、超出截尾、空列表、等宽、零窗口） |

| 常量 | 值 | 说明 |

| `MAX_WARNINGS` | 100 | 警告上限 |
| `MAX_MESSAGE_QUEUE` | 50 | 消息队列上限（满时丢最旧） |
